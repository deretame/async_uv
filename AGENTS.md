常见坑（优先规避）：

1. WSL 下监听 Windows 挂载目录（如 /mnt/c、/mnt/d）时，文件事件语义可能不稳定，可能丢事件或超时；watcher 相关测试尽量在 /tmp、/home 等 Linux 文件系统路径执行。
2. WSL 中直接在 /mnt/* 路径跑某些测试（尤其含 watcher）可能卡住或超时；可改为在 /tmp 下启动测试二进制。
3. Windows 下运行测试程序可能因运行时 DLL 路径问题报错（例如 0xC0000139）；必要时把 D:\msys2\ucrt64\bin 放到 PATH 前面再执行。
4. Alpine 容器安装依赖时网络波动较大，apk update/add 需要重试机制，并优先替换到 cernet 镜像源。
5. 长时间容器流程（拉取/编译/测试）容易被中断，尽量复用已有容器，避免每次重新拉镜像。
